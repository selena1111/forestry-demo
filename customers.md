<h1 id="removing-the-bottlenecks-of-manual-software-testing">Removing the bottlenecks of Manual Software Testing</h1>

<p>Cybercrime causes yearly financial damage close to $600 billion worldwide, nearly one percent of the global GDP (CSIS).
This damage can certainly be reduced by producing more secure and reliable software.</p>

<p>State-of-the-art software testing approaches are insufficient. For example, static analysis techniques produce a high
rate of false positives. As a result, these approaches require security experts to manually analyze the results and
identify real security issues among the large amount of generated warnings. Penetration tests, done entirely by security
experts (with some tooling support), are conducted irregularly and are usually separated from the development process.</p>

<p>The core problem is twofold: First, there are too few IT security experts, and second, testing for security and reliability issues is not automated properly.</p>

<p>Moreover, manual software testing causes unique challenges in numerous application fields such as:</p>

<h1 id="introducing-the-code-intelligence-security-suite">Introducing the Code Intelligence Security Suite</h1>

<p>We help companies solve the problems surrounding continuous software testing for security and reliability with the <em>Code
Intelligence Security Suite</em>.</p>

<p>The detection engine is based on coverage-guided fuzzing in combination with sanitizers that monitor your program&rsquo;s
behaviour at runtime. However, in contrast to expert-level Fuzzers such as <a href="http://lcamtuf.coredump.cx/afl/">AFL</a>, the
<em>CI Security Suite</em> integrates directly into the IDE of the developer, allowing her to set up and improve automated fuzz
tests in an interactive and feedback-driven manner.</p>

<p>Once the automated fuzz tests are written, our fully automated Continuous Integration component ensures that the fuzz tests are executed with every new change and new incidents are reported timely.</p>
